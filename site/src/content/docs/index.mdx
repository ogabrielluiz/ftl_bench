`ftl_bench` evaluates agents in *FTL: Faster Than Light*: a real-time-with-pause
roguelike about keeping a ship alive across combat, hazards, events, stores, and
a branching sector map. The harness turns the game into a reproducible agent
environment: observe structured state, issue a paused plan, advance the clock,
log the trajectory, and score the run.

## Why FTL is hard for agents

- **Combat is tactical and time-sensitive.** Agents must target systems, manage power, handle
  missiles and shields, repair damage, fight boarders, and decide when to run.
- **Resources are scarce.** Fuel, missiles, drone parts, scrap, hull, crew health, and reactor
  bars all trade off against each other.
- **The map is a long-horizon planning problem.** The agent chooses routes through uncertain
  events while the rebel fleet closes in.
- **Recovery matters.** Strong play means adapting after fires, oxygen loss, bad events, missed
  shots, damaged weapons, or a poor store.

## One turn

```text
OBSERVATION: compact JSON state

ACTION:
  power 3 3
  crew 1 2
  fire 0 3
  fire 1 3
  advance 150
```

The game is paused while the agent decides. The harness applies the commands, advances the
simulation, re-pauses on the next decision point, and records the result.
